# 1. 상황

1.1. 1층과 2층에 각각 turtlebot4 한대씩과 그 위에 사용자와 상호작용이 가능한 터치 스크린 UI가 존재함

1.2. 명칭상 turtlrbot4 + 노트북 거치대 + 터치스크린이 가능한 노트북을 "Alfred"라 부르겠음

1.3. 터치스크린은 각각 galaxy book3 pro 360 / galaxy book6 ultra 노트북의 터치 화면

1.4. 사용자는 터치스크린을 통해 지하철 내 존재하는 시설이 어딘지 알고 싶을 경우 현재 위치에서 해당 위치로 Alfred가 이동(일단 우린 UI파트니 이동구현은 하지 않고, JSON형식으로 서버에 데이터 전송)

1.5. 만약 사용자가 선택한 시설이 다른층에 있을 경우(1층->2층 or 2층->1층) 현재 층에 존재하는 Alfred는 에스컬레이터 또는 엘레베이터 까지 이동, 다음층에 존재하는 Alfred는 미리 에스컬레이터 또는 엘레베이터에 사용자가 도착할때까지 이동, 이후 목적지까지 이동(일단 우린 UI파트니 이동구현은 하지 않고, JSON형식으로 서버에 데이터 전송)


1.5.1. 이전 층의 Alfred의 노트북 캠을 통해 사용자의 얼굴을 저장, 에스컬레이터나 엘레베이터를 통해 이동한 사용자를 다음 층 Alfred가 사용자의 얼굴 정보를 이어받아 사용자가 맞다면 목적지까지 안내(일단 우린 UI파트니 이동구현은 하지 않고, JSON형식으로 서버에 데이터 전송) --> 해당 CV는 직접 조사해서 맞춤형 모델을 찾아서 적용할것.

# 2. UI 

2.1. 각 UI들은 본인이 1층과 2층의 Alfred임을 인지해야함. 1층용 UI와 2층용 UI가 존재
2.2. 사용자가 원하는 시설안내를 할때 이동중임을 알리는 UI가 존재
2.3. 사용자가 할 수 있는 상호작용은 "시설안내"와 "음성안내" 두개임
2.4. 시설 안내는 맵과 리스트의 목적지를 클릭하면 해당 목적지로 escort
2.5. 음성 안내는 사용자가 키오스크에 음성검색을 하면 STT로 사용자의 음성을 텍스트로 변환
2.5.1. 텍스트로 변환하여 사용자가 시설 안내(구체적으로 화장실, 출구, 안내데스크 등)를 원한다면 서버에 해당 위치를 알림
2.5.2. 사용자가 시설 안내와 관련없는 질문을 했다면 그 대답에 프리하게 응답. 서버에 알리지는 않음
2.5.3. 한국어, 영어, 일본어, 중국어 지원
