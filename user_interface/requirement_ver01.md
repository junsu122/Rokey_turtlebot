<상황>
1. Turtlebot4 위에 거치대를 얹은 노트북 2개를 설치할 예정임.
2. 노트북 2개는 samsung galaxy book이라 모니터 자체가 터치 스크린 가능.
3. 지하철 역사 내 시설 안내를 에스코트 해주는 로봇을 만들 예정임.
4. 사용자는 지하철 역사 내 시설(화장실, 역내사무실, 출구 등)을 찾기 위해 서비스를 사용함.
5. 지하철 역사는 2개의 분리된 공간(1층, 2층 - 터틀봇이 위아래로 이동할수 없다는 가정), 각 공간마다 한개의 로봇만 있기로 가정.
6. 사용자가 해당 로봇을 쓰지 않는 시간(patrol time)에는 스스로 왔다갔다 하며 위험한 물건 또는 상황이 있는지 패트롤.
7. 현재 우리는 해당 로봇위에 올라갈 키오스크 UI를 만들면 됨.

<기능 요구사항>
1. UI 실행시 크롬 브라우저 기준으로 맨 위에 존재하는 탭 표시줄(tab bar), 제목 표시줄(title bar), 주소 표시줄(omnibox),  북마크 바와 맨 아래 존재하는 Windows 작업 표시줄(taskbar)이 전부 없는 키오스크 형식으로 나와야함.
2. 사용자가 해당 로봇을 쓰지 않는 시간(patrol time)인 경우 UI는 키오스크 형식에 웃는(^^) 얼굴 표정 전체로 나올것. 맨 아래 글씨로 "사용할거면 아무키나 누르세요" 하나만 적을것. 
3. 사용자가 터치스크린에 터치를 해서 patrol time을 멈추면 사용자 초기 UI에는 아주 크게 노출된 버튼 2개(가명 A, B)를 중앙에 설치. 맨 아래 오른쪽에 [직원 호출] 버튼 설치.
4. 버튼 A는 사용자에게 맵을 띄워서 이 공간(1층 또는 2층)의 시설들이 건설도면처럼 나와있고, 해당 시설을 클릭하면 서버에서 ros토픽을 통해 turtlebot이 해당 장소로 이동함.
5. 버튼 B는 사용자와 STT를 통해 LLM으로 시설을 도출해, 해당 시설을 서버에서 ros토픽을 통해 turtlebot이 해당 시설로 이동함.
6. 만약 사용자가 선택한 시설이 다른 층으로 올라가야하는 시설인 경우, 첫 공간에서 계단, 에스컬레이터, 엘레베이터 중에 하나로 이동하고 다음 공간에서 다른 터틀봇이 대기를 할것. 이후 다음 공간에 존재하는 터틀봇이 시설까지 안내.
7. 사용자의 시설 안내를 통해 이동 중인 경우(2개 층을 이동하는 경우 터틀봇 둘다 해당) UI는 키오스크 형식에 웃는(^^) 얼굴 표정 전체로 나오면서 "시설 안내중" 문구 추가.
8. UI의 텍스트와 버튼은 최대한 크게 만들것.
9. 버튼 A와 B의 텍스트 이름은 짧은 단어로 각각 만들어낼것. 시설안내? 음성안내?
10. 사용자 요구사항에 따른 시설 안내가 끝나면 다시 patrol time으로 돌아감.
11. [직원 호출] 버튼을 누르면 팝업창과 함께 직원이 오고 있습니다 문구가 뜨면서 직원이 닫기 버튼을 눌러 팝업창이 닫힘.
